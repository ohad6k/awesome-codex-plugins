# Run Link Reclamation Workflow

Use this workflow to find, qualify, request, and verify attribution links.

## When to Use

- The user has unlinked mentions, lost links, or image uses.
- A brand, report, tool, data source, or old URL needs auditing.
- Existing coverage needs follow-up.

## Prerequisites

- Brand names, people names, product names, asset URLs, and historical URLs.
- Backlink or mention exports when available.
- Preferred destination URLs.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Build Query Set

- [ ] List brand, product, executive, report, tool, image, and campaign names.
- [ ] Add historical URLs and known asset filenames.
- [ ] Include misspellings and alternate brand names.

### Step 2: Discover Opportunities

- [ ] Search web, mention tools, backlink exports, image search, and crawl data.
- [ ] Deduplicate by domain and page.
- [ ] Capture context, current link status, and contact path.

### Step 3: Qualify And Prioritize

- [ ] Apply hard rejects for spam, irrelevance, negative context, and unstable destination.
- [ ] Score legitimate opportunities by trust, relevance, attribution clarity, and ease.
- [ ] Fix owned redirects and canonicals first.

### Step 4: Draft Requests

- [ ] Use evidence-backed attribution language.
- [ ] Include publisher URL and exact destination URL.
- [ ] Avoid threats or SEO language for normal outreach.

### Step 5: Send, Track, And Verify

- [ ] Track owner, send date, follow-up, response, and result.
- [ ] Verify live link status, attributes, and destination.
- [ ] Route new relationships or coverage gaps to future campaigns.

## Exit Criteria

Task is complete when:

- [ ] Opportunities are bucketed as request, fix internally, monitor, or reject.
- [ ] Requests are evidence-backed and low friction.
- [ ] Live outcomes are verified, not assumed.
