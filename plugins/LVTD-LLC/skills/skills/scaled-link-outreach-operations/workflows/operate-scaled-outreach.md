# Operate Scaled Outreach Workflow

Use this workflow to plan, run, and diagnose scaled link outreach.

## When to Use

- The user has a large prospect list.
- A proven outreach offer needs scaling.
- A campaign has low replies, bounces, complaints, or poor placements.

## Prerequisites

- Linkable asset and offer.
- Prospect list with evidence fields.
- Sending platform, suppression list, and tracking owner.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Define The Operating Model

- [ ] State offer, segment, send volume, geography, and compliance constraints.
- [ ] Identify manual-review segments and no-send categories.

### Step 2: Prepare The List

- [ ] Deduplicate domains and contacts.
- [ ] Verify contact data where appropriate.
- [ ] Apply suppression, opt-out, and prior-contact rules.
- [ ] Confirm each row has evidence for the ask.

### Step 3: Build Templates And Variables

- [ ] Create one template per segment.
- [ ] Use variables that change the substance of the pitch.
- [ ] Review subject lines for accuracy and spam risk.

### Step 4: Pilot

- [ ] Send a representative pilot.
- [ ] Monitor bounces, complaints, replies, and qualitative feedback.
- [ ] Decide stop, revise, or expand based on thresholds.

### Step 5: Expand And Monitor

- [ ] Increase volume gradually.
- [ ] Track contacts, replies, placements, opt-outs, and relationship notes.
- [ ] Route high-value replies to manual handling.

### Step 6: Report And Learn

- [ ] Report by segment, not only aggregate.
- [ ] Recommend next experiment or stop condition.

## Exit Criteria

Task is complete when:

- [ ] Segments, templates, tools, and owners are explicit.
- [ ] Pilot evidence supports any expansion.
- [ ] Compliance and suppression handling are ready.
