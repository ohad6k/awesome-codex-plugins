# Linkable Content Architecture

Use this skill to make linkable assets findable, crawlable, internally connected, and useful before promotion or outreach begins.

This skill is derived from Garrett French and Eric Ward's *Ultimate Guide to Link Building* and uses transformed guidance with durable book-topic references. Do not copy book prose into user outputs.

## Quick Start

1. Use the Contents table below to choose the smallest useful reference set.
2. Use `workflows/audit-linkable-content-architecture.md` for a full architecture audit.
3. Check whether the asset is orphaned, isolated, unstable, blocked, or weakly linked.
4. Improve internal links, hubs, canonicals, redirects, and conversion paths before outreach.
5. Return a prioritized fix plan with verification steps.

## Default Output

When asked about linkable content architecture, return:

1. **Asset map** - target assets, hubs, business pages, supporting pages, and URL status.
2. **Architecture findings** - crawlability, search index status, orphan risk, internal links, canonicals, redirects, and navigation.
3. **Link equity plan** - inbound link destinations, hub links, contextual links, and commercial-path support.
4. **Promotion readiness** - whether outreach should proceed, pause, or point to a stronger URL.
5. **Tool choices** - site crawler, backlink exports, analytics/search data, CMS, log files, and internal-link visualization.
6. **Verification** - crawl checks, link checks, index status, analytics events, and post-launch monitoring.

## Core Posture

- Do not promote assets that are technically hidden or internally disconnected.
- External links work harder when the destination is part of a clear site structure.
- Internal links should connect linkable assets to useful next steps without burying the asset.
- Stable URLs and clean canonicals reduce wasted outreach.
- Architecture fixes should happen before large outreach pushes.
