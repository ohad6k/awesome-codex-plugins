# Audit Linkable Content Architecture Workflow

Use this workflow before promoting a linkable asset.

## When to Use

- A campaign is about to promote a report, tool, guide, template, or resource hub.
- Existing assets have links but weak SEO or business impact.
- A site redesign, migration, or content cleanup may affect promoted URLs.

## Prerequisites

- Target URLs and intended campaign URLs.
- Crawl data or permission to crawl.
- Backlink, analytics, and Search Console exports when available.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Map Assets And Paths

- [ ] List target assets, hubs, related pages, commercial pages, and current URLs.
- [ ] Identify campaign URLs and canonical destinations.

### Step 2: Run Technical Checks

- [ ] Check status codes, redirects, canonicals, robots/meta directives, mobile usability, and page speed risks.
- [ ] Confirm analytics and conversion events if measurement matters.

### Step 3: Audit Internal Links

- [ ] Find incoming internal links and outgoing links for each asset.
- [ ] Mark orphan, silo, and weak-anchor issues.
- [ ] Add hub, contextual, and next-step link recommendations.

### Step 4: Connect Link Equity To Business Paths

- [ ] Define useful paths from asset to supporting and commercial pages.
- [ ] Avoid aggressive conversion elements that reduce citation value.

### Step 5: Decide Promotion Readiness

- [ ] Bucket each asset as ready, fix before outreach, redirect/rebuild, or do not promote.
- [ ] Define verification commands or tool checks for fixes.

## Exit Criteria

Task is complete when:

- [ ] Every promoted URL is stable, crawlable, searchable, and internally linked.
- [ ] Internal link fixes are specific and verifiable.
- [ ] Outreach destination choices match architecture evidence.
