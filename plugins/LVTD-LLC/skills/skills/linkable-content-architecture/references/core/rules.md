# Linkable Content Architecture Rules

Use these rules when auditing assets before link promotion.

## Core Rules

### 1. Verify Technical Access

- Check status code, canonical target, robots/meta directives, redirects, and search index status.
- Do not promote blocked, duplicate, or unstable URLs.

### 2. Remove Orphan And Silo Problems

- Link assets from relevant hubs, navigation, resource pages, or contextual articles.
- Link from the asset to next-step resources and commercial pages where useful.

### 3. Keep Link Targets Stable

- Use durable URLs for outreach.
- Avoid campaign parameters as the canonical destination.
- Create redirects before replacing or moving promoted assets.

### 4. Preserve Asset Usefulness

- Do not bury the useful content behind forms, popups, or aggressive conversion paths.
- Let the asset satisfy the publisher's audience first.

### 5. Measure After Promotion

- Track inbound links, referral traffic, engagement, assisted conversions, and internal-link performance.
- Watch for broken links after redesigns or migrations.

## Red Flags

- Asset is not linked from anywhere on the site.
- Canonical points away from the promoted URL.
- Outreach URL redirects through a long chain.
- Asset links only to sales pages and no supporting resources.
- The page is technically live but not useful on mobile.
