# Evaluate Directory Links Workflow

Use this workflow to review directory and listing opportunities.

## When to Use

- The user has a list of directories or citation sites.
- A directory submission is being considered.
- Existing directory backlinks need quality review.

## Prerequisites

- Candidate directory URLs.
- Business category, geography, and target URL.
- Budget or paid-placement constraints.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Classify Candidates

- [ ] Tag each candidate by directory type.
- [ ] Note geography, category, niche, and paid/free status.

### Step 2: Apply Hard Rejects

- [ ] Reject generic, low-quality, irrelevant, abandoned, or link-selling directories.
- [ ] Flag paid followed links for risk review.

### Step 3: Inspect Quality

- [ ] Check audience value, editorial standards, listing depth, neighboring listings, and recency.
- [ ] Check search index status, link attributes, destination rules, and category fit.

### Step 4: Recommend Action

- [ ] Bucket as submit, update, monitor, reject, or remove/disavow review.
- [ ] Explain evidence and tool checks for each decision.

### Step 5: Track Submission

- [ ] Record credentials, cost, renewal, status, destination, and review date.
- [ ] Verify live listing and link attributes after approval.

## Exit Criteria

Task is complete when:

- [ ] Every candidate has a decision and evidence.
- [ ] Paid or risky links are clearly flagged.
- [ ] Approved listings have tracking and verification steps.
