# Audit Sales Page Citability Workflow

Use this workflow to decide whether a commercial page can earn legitimate editorial links.

## When to Use

- A campaign wants links to a product, service, pricing, or feature page.
- Guest posts or resource pitches need a natural citation target.
- A commercial page needs reference-worthy sections before outreach.

## Prerequisites

- Target URL and business goal.
- Intended publisher or outreach context.
- Access to page content, analytics, and user questions when available.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Define Citation Context

- [ ] Name the publisher type and reader task.
- [ ] Identify why a link to this page would help that reader.
- [ ] Mark the page as not-ready if no legitimate citation scenario exists.

### Step 2: Map The Action Cycle

- [ ] List what the audience needs before, during, and after the buying/problem moment.
- [ ] Pull support, sales, review, and search-query evidence when available.

### Step 3: Audit Current Page

- [ ] Check citable elements, section anchors, evidence, crawlability, and internal links.
- [ ] Identify promotional claims that need proof or should not be pitched.

### Step 4: Recommend Modules

- [ ] Choose definitions, data, tools, examples, templates, FAQs, or criteria.
- [ ] Specify section titles, anchors, proof sources, and owner.
- [ ] Decide whether a separate support asset is better.

### Step 5: Prepare Outreach Readiness

- [ ] Create citation-safe phrasing.
- [ ] Confirm page status, canonical, search index status, and internal links.
- [ ] Return ready, needs changes, or do-not-pitch.

## Exit Criteria

Task is complete when:

- [ ] Each recommended link target has a reader-serving reason.
- [ ] Page changes are specific enough for a writer or CMS owner.
- [ ] Outreach readiness is based on verified page evidence.
