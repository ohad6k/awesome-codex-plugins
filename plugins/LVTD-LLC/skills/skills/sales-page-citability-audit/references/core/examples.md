# Sales Page Citability Audit Examples

Use these examples to turn a commercial page into a more credible link target.

## Citable Module Ideas

| Page Type | Add This | Why It Helps |
|-----------|----------|--------------|
| SaaS feature page | Calculator, benchmark, or implementation checklist | Gives writers a practical reference |
| Service page | Process framework and qualification criteria | Helps readers evaluate options |
| Product category page | Glossary, comparison table, and buying criteria | Supports educational citations |
| Pricing page | Cost model, assumptions, and FAQ | Lets publishers cite transparent pricing logic |
| Case study page | Methods, before/after data, and constraints | Makes evidence more credible |

## Audit Finding Pattern

```
Finding: The page asks for guest-post links but has no citable section.
Evidence: No definitions, data, examples, tools, or section anchors.
Fix: Add "Implementation checklist" with anchor #implementation-checklist and internal links from two related guides.
Pitch angle: Cite the checklist when explaining how teams evaluate [topic].
```

## Citation Language

| Weak | Better |
|------|--------|
| "Link to our product page" | "Cite the public checklist section for readers comparing implementation steps" |
| "Mention our pricing" | "Reference the cost model and assumptions section" |
| "Use our page as a source" | "Use the methodology-backed benchmark chart as the source" |

## When To Use A Support Asset Instead

- The sales page cannot carry long educational material without hurting conversion.
- The citation target needs methodology or data that deserves its own URL.
- The publisher would never cite a vendor page in that context.
