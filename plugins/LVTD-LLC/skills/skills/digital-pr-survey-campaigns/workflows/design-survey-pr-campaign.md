# Design Survey PR Campaign Workflow

Use this workflow to plan data-led link campaigns before survey launch or outreach.

## When to Use

- The user wants a linkable survey, study, or expert panel.
- A dataset exists but needs PR framing.
- A report page or journalist pitch needs review.

## Prerequisites

- Brand, topic, and business goal.
- Intended audience or publisher segment.
- Data source or budget for collection.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Define Story And Linker Audience

- [ ] Write 3-5 possible headlines.
- [ ] Name reporter beats and publisher types.
- [ ] Reject angles that do not serve an external audience.

### Step 2: Design Data Collection

- [ ] Choose sample source, respondent criteria, and target size.
- [ ] Draft questions that avoid leading or compound wording.
- [ ] Define field dates, exclusions, and quality controls.

### Step 3: Analyze And Select Findings

- [ ] Clean and segment responses.
- [ ] Find findings that are surprising, useful, or timely.
- [ ] Record limitations beside every major claim.

### Step 4: Package The Asset

- [ ] Create report structure, charts, methodology, and citation language.
- [ ] Add section anchors for major findings.
- [ ] Confirm the URL is crawlable, stable, and internally linked.

### Step 5: Plan Outreach

- [ ] Segment reporters by beat and angle.
- [ ] Decide whether any exclusive or embargo is warranted.
- [ ] Draft pitches with one strong finding and methodology proof.

### Step 6: Measure And Reclaim

- [ ] Track coverage, links, unlinked mentions, referral traffic, and relationships.
- [ ] Route unlinked coverage to link reclamation.

## Exit Criteria

Task is complete when:

- [ ] Claims match the data and methodology.
- [ ] The report page is citable before outreach.
- [ ] Pitch segments, tools, and tracking fields are defined.
