# Договор артефактов жизненного цикла клиента

Каркас создаёт пустой проект со статусом `unverified`. Пустой шаблон не является подтверждённым исследованием и не может служить основанием для рекламы или записи в кабинет.

## Обязательные файлы

1. `client-kb.md` — база знаний клиента с разделением `confirmed`, `inferred`, `unknown`.
2. `source-register.tsv` — реестр каждого источника и его доказательства.
3. `product-map.md` — карта продукта, синонимов, защищённых слов и ограничений.
4. `routing-map.tsv` — намерение, домен, точная страница выдачи, точная страница сайта, кампания и группа в отдельных полях.
5. `landing-inventory.tsv` — машинный реестр посадочных страниц.
6. `fact-check-log.tsv` — журнал подтверждений, противоречий и просроченных фактов.
7. `human-review.tsv` — очередь решений человека без автоматического утверждения.
8. `access-status.md` — только состояния доступов; без ключей, токенов и содержимого файлов доступа.
9. `competitor-raw-register.tsv` — полный исходный сбор без смыслового сокращения.
10. `proposal-pack.md` — выводы, связанные с реестром источников.
11. `research/jobs/*` — выключенные по умолчанию задания сбора.
12. `research/raw/*` — неизменённые ответы и их контрольные суммы.
13. `research/analysis/*` — ручной анализ, покрытие и нерешённые вопросы.
14. `.codex/yandex-performance-client.json` — локальные настройки без секретов и рабочих значений по умолчанию.

## Реестр источников

Каждая строка содержит `source_id`, `source_type`, `source_url`, `captured_at`, `status`, `evidence_path`, `reviewer`, `reviewed_at`, `expires_at`, `reversal_ref`, `artifact_sha256` и `notes`.

- `status=unverified` — исходное состояние шаблона;
- `status=confirmed` допустим только при непустых источнике, времени, доказательстве и проверяющем;
- противоречие хранится отдельной строкой и не перезаписывает исходный факт;
- абсолютные домашние пути и закрытые сведения в переносимые отчёты не входят.

## Правила обновления

1. Новая выгрузка всегда получает отдельную строку и контрольную сумму.
2. Конкретная страница выдачи не заменяется доменом; домен используется только для группировки.
3. Конкретная посадочная страница не заменяется корнем сайта.
4. Намерение, кампания и группа не объединяются в одно название.
5. Просроченный факт возвращается в `unverified` до новой проверки.
6. Все смысловые кандидаты проходят ручную проверку до формирования пакета записи.
