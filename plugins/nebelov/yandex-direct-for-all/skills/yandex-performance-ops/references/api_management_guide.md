# Безопасное управление через API Директа

Официальные источники: [заголовки запросов и ответов](https://yandex.ru/dev/direct/doc/ru/concepts/headers), [ограничения и баллы](https://yandex.ru/dev/direct/doc/ru/concepts/units), [изменение групп](https://yandex.ru/dev/direct/doc/ref-v5/adgroups/update.html).

## Чтение

Универсальный мост принимает только `get` для закрытого списка служб. Токен и `Client-Login` читаются из локальной среды, а не из аргументов инструмента. Для единых перфоманс-кампаний и групп используется `v501`, когда этого требует описание конкретного метода.

## Запись

Единственный разрешённый вход — `direct_write_gate.py`. Он принимает пакет и отдельный файл его контрольной суммы. До любого сетевого вызова проверяются среда, область клиента, отдельный токен записи, вооружение, срок, подтверждение владельца, общий бюджет баллов, верхний предел баллов каждой операции и свободная блокировка. Сумма верхних пределов операций не может превышать общий бюджет; превышение подтверждённого предела фактическим ответом немедленно останавливает пакет.

Разрешены только перечисленные в схеме пары службы и метода. Каждый ответ учитывает заголовок `Units`, который Яндекс описывает как «израсходовано / доступный остаток / суточный лимит». При исчерпании утверждённого бюджета следующая запись не отправляется.

Яндекс допускает не более пяти одновременных запросов от одного рекламодателя; шлюз выполняет записи последовательно. Заголовок `Use-Operator-Units` не добавляется автоматически, поэтому пакет не может молча переключить списание на агентство.

После каждой записи выполняется `get` утверждённой области. Частичный ответ, ошибка строки или несовпадение перечитанного состояния оставляет операцию незавершённой и блокирует зависимые действия.
