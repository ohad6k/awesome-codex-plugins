# Поисковые запросы и кандидаты в минус-фразы

1. Соберите полный отчёт за указанный период и проверьте манифест каждого фрагмента.
2. Сохраните каждую исходную строку и её контрольную сумму.
3. Для каждого кандидата заполните источник, доказательство и область действия.
4. Не переносите прежнее решение на новую строку: известное минус-слово даёт только отметку `pending_review`.
5. Ручной проверяющий заполняет решение, имя, время, срок и ссылку восстановления.
6. Пакет записи строится только из строк с действующим ручным решением и проходит общий шлюз.

Автоматическое сокращение очереди и механическое превращение запросов в минус-слова запрещены.
