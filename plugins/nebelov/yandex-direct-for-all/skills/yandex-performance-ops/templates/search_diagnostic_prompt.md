# Диагностика Поиска и РСЯ

Шаблон предназначен для чтения сохранённых данных и подготовки кандидатов ручной проверки. Он не принимает окончательные смысловые решения и не выполняет запись.

## Перед анализом

1. Зафиксировать период, часовой пояс, валюту, цели, модель атрибуции и область кампаний.
2. Собрать данные через штатные сценарии чтения без передачи токена аргументом.
3. Проверить состояния постраничности, число страниц и строк, признак завершения и контрольные суммы.
4. Разделить Директ, Метрику, аналитический отчёт, сырые сделки и коллтрекинг по отдельным источникам.
5. При неполном обязательном источнике продолжить только как неполную диагностику без утверждённых выводов.

## Поиск

- сравнить период с выбранной базой, не подменяя её выдуманным порогом;
- пройти каждую строку отчёта поисковых запросов;
- сохранить целевые и нецелевые признаки, защищённые слова и конфликтующие намерения;
- для минус-фразы указать точный запрос, область действия и риск отсечь целевой спрос;
- любые изменения ключей, автотаргетинга, текстов и посадочных страниц оставить кандидатами.

## РСЯ

- анализировать каждую кампанию и площадку отдельно;
- не считать отсутствие конверсий, вид домена или высокий показатель достаточным решением;
- сохранить исходную строку, период, цель, показатели и текущий список запрещённых площадок;
- площадки с подтверждёнными конверсиями отмечать как защищённые от автоматического запрета.

## Выход

1. Подтверждённые факты и полнота источников.
2. Причинные связи, которые доказаны данными; предположения помечены отдельно.
3. Полная таблица поисковых запросов и площадок без удаления строк.
4. Кандидаты на изменения с полями ручного решения из `tasks_format.md`.
5. Ожидаемый эффект только как воспроизводимый расчёт с явными допущениями.

Для применения утверждённых изменений создаётся отдельный пакет `direct_write_pack.schema.json` и используется `direct_write_gate.py`.
