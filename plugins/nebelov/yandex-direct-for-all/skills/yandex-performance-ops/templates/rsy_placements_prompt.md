# Кандидаты по площадкам РСЯ

Задача: для каждой кампании отдельно сформировать полный перечень проверяемых кандидатов. Шаблон ничего не отключает и не создаёт пакет записи.

## Источники

- завершённый отчёт Директа по площадкам за явно указанный период;
- текущий список запрещённых площадок из снимка управления;
- цель, модель атрибуции, часовой пояс и фильтры отчёта;
- контекст кампании из аналитических систем, если он доступен.

Данные другой кампании не используются как доказательство. Сырой токен, код авторизации и логин в шаблон не вставляются.

## Проверка каждой строки

1. Сохранить понятное название кампании, площадку, период и исходные показатели.
2. Проверить конверсии по выбранной цели; строка с подтверждённой конверсией не может быть кандидатом на запрет без отдельного обоснования владельца.
3. Зафиксировать наблюдаемый сигнал: расход, клики, долю переходов, отказы, аномалию или тип площадки.
4. Не считать тип домена, отсутствие конверсий либо формульный порог самостоятельным решением.
5. Проверить наличие площадки в текущем списке и доступное число мест по действующим ограничениям.
6. Оставить `decision` пустым; ротацию существующего списка также оформлять отдельными кандидатами.

## Выход

Одна строка на каждую рассмотренную площадку:

- `source`, `source_row_sha256`, `evidence`, `scope`;
- понятные названия кампании и площадки;
- период, цель, модель атрибуции и показатели;
- `signal`, `already_excluded`, `candidate_status=pending_review`;
- пустые `reviewer`, `reviewed_at`, `expires_at`, `decision`, `reversal_ref`.

Отдельно укажите полноту исходного отчёта. Если она не доказана, весь результат имеет статус `incomplete`.
