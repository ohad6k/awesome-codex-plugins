# Проверка соответствия текстов группе на Поиске

Задача: сформировать варианты текстов для ручной проверки. Ни один вариант не считается готовым к публикации и не передаётся в запись автоматически.

## Вход

- карта групп и их намерений;
- текущие тексты объявлений;
- карта продукта, защищённые слова и допустимые посадочные страницы;
- исходный снимок с контрольной суммой.

## Правила

1. Проверять каждую группу отдельно по её устойчивому `group_uid`.
2. Для каждого вывода сохранять исходный текст, намерение группы и конкретное доказательство.
3. Не объявлять лексическое совпадение смысловой релевантностью без ручной проверки.
4. Не подменять требования к количеству объявлений выдуманным постоянным числом: использовать действующий договор проекта и официальные ограничения.
5. Не переносить решение между группами и не создавать пакет записи.

## Выход

Таблица кандидатов содержит:

- `group_uid`;
- понятные названия кампании и группы;
- исходный текст и его контрольную сумму;
- предлагаемый вариант;
- доказательство соответствия намерению;
- найденные конфликты и дубли;
- `candidate_status=pending_review`;
- пустые `reviewer`, `reviewed_at`, `expires_at`, `decision`, `reversal_ref`.

Каждая входная группа должна присутствовать в выходе, включая группы без найденного варианта.
