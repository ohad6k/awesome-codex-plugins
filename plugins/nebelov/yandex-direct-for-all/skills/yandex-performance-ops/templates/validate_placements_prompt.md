# Независимая проверка кандидатов по площадкам РСЯ

Проверяющий не создавал кандидатов и не меняет исходный список. Его задача — проверить доказательства и вернуть замечания для ручного решения. Даже успешная проверка не означает автоматического отключения площадки.

## Вход

- кандидаты для одной явно названной кампании;
- завершённый исходный отчёт по площадкам с периодом, целью и моделью атрибуции;
- текущий список запрещённых площадок;
- контрольные суммы исходных строк и состояние полноты.

## Проверки

1. Каждая строка кандидата находится в исходном отчёте и имеет совпадающую контрольную сумму.
2. Период, цель, модель атрибуции, фильтры и кампания совпадают.
3. Подтверждённые конверсии и иные защищающие сигналы не скрыты.
4. Формульный порог, тип домена и отсутствие конверсий представлены только как признаки, а не как автоматический вердикт.
5. Кандидат не взят из другой кампании и не дублирует уже запрещённую площадку.
6. Предложение не превышает доступное число мест по действующим ограничениям.
7. Для ротации сохранено отдельное доказательство по каждой удаляемой и добавляемой площадке.
8. Поля `reviewer`, `reviewed_at`, `expires_at`, `decision` и `reversal_ref` не были автоматически заполнены автором кандидата.

## Выход

- `PASS`: доказательства полны, кандидат допускается к ручному решению;
- `FAIL`: доказательства противоречат источнику или неполны;
- `NEEDS_DATA`: обязательный источник не завершён.

Для каждой строки перечислите понятное название площадки, результат проверки, доказательство и необходимые исправления. Не создавайте пакет записи и не вызывайте интерфейс записи.
