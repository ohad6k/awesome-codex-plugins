# Задание ручной смысловой проверки

Проверяющий получает полный набор кандидатов без сокращения. Для каждой строки он обязан видеть исходную строку, контрольную сумму, источник, доказательство и область действия.

Проверяющий заполняет:

- `reviewer` — понятное имя ответственного;
- `reviewed_at` и `expires_at`;
- `decision` — `approve`, `reject` либо `needs_context`;
- `reversal_ref` — ссылка на способ восстановления для утверждённого изменения;
- пояснение, относящееся именно к этой строке.

Совпадение со старым правилом, словом или площадкой не переносит прежнее решение. Оно остаётся доказательством-кандидатом. Нельзя удалять строки очереди, объединять разные исходные строки под одной контрольной суммой или автоматически заполнять окончательное решение.
