# Восстановление доступа

1. Определите службу: Директ, Метрика, Wordstat или Поиск Яндекса. Не подставляйте один вид доступа вместо другого.
2. Проверьте, что локальный файл существует, принадлежит текущему пользователю и имеет права `0600`; каталог должен иметь `0700`.
3. Для Директа проверьте выбранную среду и обязательный логин клиента. Песочный и рабочий токены не взаимозаменяемы.
4. Для Директа запустите `scripts/start_yandex_user_auth.sh --service direct --client-login 'логин-рекламодателя'`; для Метрики — `scripts/start_yandex_user_auth.sh --service metrika`. Встроенные опубликованные приложения не требуют регистрации собственного приложения.
5. Не передавайте токен, код или секрет через аргументы команды, журналы, снимки экрана и обращения в поддержку.
6. После обмена дождитесь обязательной сверки приложения и чтения службы. Один успешный обмен токена без чтения не доказывает доступ.
7. Для Директа выполните `check_access_paths.py --access-file ...`; для сборщика проверьте `complete=true` во всех обязательных манифестах.
8. При ошибке сохраните только обезличенные код ответа, название службы и время. Не сохраняйте заголовок `Authorization`, содержимое файла доступа и абсолютный домашний путь.

Если доступ отозван, повторите авторизацию. Не исправляйте проблему публикацией нового токена в репозитории.
