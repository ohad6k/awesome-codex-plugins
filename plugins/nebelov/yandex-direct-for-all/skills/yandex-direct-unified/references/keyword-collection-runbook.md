# Сбор и проверка ключевых фраз

1. Создать таблицу продуктовых направлений и маршрутов на посадочные.
2. Для каждого направления выписать синонимы, действия, коммерческие признаки, варианты языка и написания.
3. Сохранить происхождение каждой маски до обращения к Wordstat.
4. Собрать Wordstat v2 по целевой географии: topRequests, associations, regions, при необходимости dynamics.
5. Добавить поисковые фразы из подтверждённого кабинета и результаты поиска с сохранением источника.
6. Присвоить каждой исходной строке candidate_id; не удалять строки до решения.
7. Разбирать строки последовательными частями. Для каждой указать смысл, сегмент, посадочную, решение и причину.
8. Отдельно проверить защищённые слова перед предложением минус-слов.
9. Сопоставить все исходные строки с таблицами включения, исключения и отложенных решений.
10. Создать отчёт о непокрытых целевых запросах. Любой необъяснённый разрыв блокирует сборку.

Допустимые сценарии: сбор исходных ответов, нормализация формата без потери строк, нумерация, разбиение, объединение, проверка схемы и полноты. Запрещены автоматический смысловой отбор, оценка коммерции, готовый пакет ключей и автоматический статус покрытия.
